# Chronicle Website Launch Plan

## Recommended Stack

Use a single static page hosted on Cloudflare Pages, with downloadable app files hosted separately on GitHub Releases.

Why:

- free static hosting is enough for Chronicle's first public site
- simple deployment from a local folder or Git repo
- no backend required
- easy to replace later with a fuller site
- GitHub Releases is a practical free place to host the Mac and Windows build files

## Recommended First Domain/Email Setup

Current public contact addresses:

- `hello.chronicle.app@gmail.com`
- `chronicle.app+support@gmail.com`
- `chronicle.app+press@gmail.com`

If a custom domain is not ready yet, Chronicle can launch first with dedicated Gmail addresses and move to domain mail later.

## Recommended First-Page Sections

1. Hero
2. What Chronicle is
3. Who it is for
4. Why Chronicle is different
5. Pricing
6. Evaluation policy
7. Privacy / review disclaimer
8. Downloads
9. Contact / licensing inquiry

## Accessibility Note

Cloudflare Pages can host an accessible site, but it does not guarantee that a page is accessible by itself.

Chronicle's accessibility responsibility still sits in the page we ship:

- semantic HTML
- correct heading order
- strong color contrast
- keyboard usability
- meaningful link text
- responsive layout
- manual testing with screen readers and browser zoom

## First Launch Copy Decisions

- emphasize hard-document recovery
- emphasize accessibility and review workflows
- give people an immediate Mac and Windows download path
- keep pricing visible
- keep evaluation terms visible
- avoid donationware language on the sales page
- make it clear that Chronicle reduces cleanup, not human verification
