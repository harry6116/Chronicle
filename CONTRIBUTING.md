# Contributing to Chronicle

Thank you for your interest in improving Chronicle.

## Guidelines

- Maintain accessibility-first principles.
- Preserve semantic output structure.
- Avoid introducing non-ASCII encoding artifacts.
- Keep chunking safeguards intact.
- Clearly document any changes to prompt logic.

## Submitting Changes

1. Fork the repository.
2. Create a feature branch.
3. Submit a pull request with a clear explanation.

All contributions must be compatible with GPLv3.

By submitting code, you agree that your contribution may be redistributed under GPLv3.
